# Tasks API

The Tasks API allows you to create, manage, and monitor individual tasks within your execution runs. Tasks represent individual units of work assigned to agents/pilots.

## Get Tasks by Agent Handle

Fetch all tasks assigned to a specific agent/pilot using their handle. This returns full task metadata including user info, input payloads, attachments, and execution details.

```http
GET /api/v2/platform/agents/{handle}/tasks/
```

### Path Parameters

| Name   | Type   | Required | Description                              |
|--------|--------|----------|------------------------------------------|
| handle | string | Yes      | Unique handle/identifier of the agent    |

### Headers

| Name          | Type   | Required | Description                     |
|---------------|--------|----------|---------------------------------|
| Authorization | string | Yes      | JWT token format: `JWT <token>` |
| Content-Type  | string | Yes      | `application/json`              |

### Example Request

```http
GET /api/v2/platform/agents/sales-agent-v1/tasks/
Headers:
  Authorization: JWT eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  Content-Type: application/json
```

## Get Tasks by Space Token

Fetch all tasks for a specific space using the space_token. Supports pagination with page and query parameters for efficient data retrieval.

```http
GET /api/v2/platform/spaces/{space_token}/tasks/
```

### Path Parameters

| Name        | Type   | Required | Description                            |
|-------------|--------|----------|----------------------------------------|
| space_token | string | Yes      | Public token identifying the workspace |

### Query Parameters

| Name      | Type    | Required | Description                          |
|-----------|---------|----------|--------------------------------------|
| page      | integer | No       | Page number (default: 1)             |
| page_size | integer | No       | Number of tasks per page (default: 20) |

### Headers

| Name          | Type   | Required | Description                     |
|---------------|--------|----------|---------------------------------|
| Authorization | string | Yes      | JWT token format: `JWT <token>` |
| Content-Type  | string | Yes      | `application/json`              |

### Example Request

```http
GET /api/v2/platform/spaces/space_abc123xyz/tasks/?page=1&page_size=20
Headers:
  Authorization: JWT eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  Content-Type: application/json
```

## Create a Task in a Space

Create a new task inside a given space using the space's public token. You can assign the task to a pilot/agent, attach multiple documents, provide additional context, and optionally schedule the execution.

```http
POST /api/v2/platform/spaces/{space_token}/tasks/
```

### Path Parameters

| Name        | Type   | Required | Description                                      |
|-------------|--------|----------|--------------------------------------------------|
| space_token | string | Yes      | Public token of the space where task is created  |

### Headers

| Name          | Type   | Required | Description                     |
|---------------|--------|----------|---------------------------------|
| Authorization | string | Yes      | JWT token format: `JWT <token>` |
| Content-Type  | string | Yes      | `application/json`              |

### Request Body

| Field     | Type   | Required | Description                                           |
|-----------|--------|----------|-------------------------------------------------------|
| pilot     | string | Yes      | Agent/pilot handle to assign the task                 |
| documents | array  | Yes      | Array of document objects with contact/task details   |
| context   | string | No       | Additional context or instructions for the task       |
| schedule  | object | No       | Scheduling configuration (`type`: `now` or `later`)   |

### Document Object Fields

| Field            | Type   | Description                              |
|------------------|--------|------------------------------------------|
| name             | string | Contact name                             |
| email            | string | Contact email address                    |
| number           | string | Primary contact number                   |
| contact_number   | string | Contact phone number                     |
| alternate_number | string | Alternate phone number                   |
| context          | string | Task-specific context                    |
| labels           | array  | Array of labels/tags                     |
| title            | string | Document title                           |
| description      | string | Document description                     |
| document_id      | string | Unique document identifier               |
| created          | string | Document creation timestamp (ISO 8601)   |

### Example Request


## Common Error Codes

| Status Code | Description                                      |
|-------------|--------------------------------------------------|
| 200         | Success - Request completed successfully         |
| 206         | Partial Content - Business logic error occurred  |
| 400         | Bad Request - Invalid parameters provided        |
| 401         | Unauthorized - Invalid or missing JWT token      |
| 403         | Forbidden - Access denied to the resource        |
| 404         | Not Found - Agent or Space not found             |
| 500         | Internal Server Error - Server-side error        |

```python Python
import requests

def fetch_tasks_by_agent(agent_handle: str):
    """Get tasks by agent handle"""
    url = f'https://unpod.dev/api/v2/platform/agents/{agent_handle}/tasks/'

    headers = {
        'Authorization': 'JWT your-jwt-token',
        'Content-Type': 'application/json'
    }

    response = requests.get(url, headers=headers)
    data = response.json()

    print(f"Total tasks: {data['count']}")
    for task in data['data']:
        print(f"Task {task['task_id']}: {task['status']} - {task['input']['name']}")

def fetch_tasks_by_space(space_token: str, page: int = 1, page_size: int = 20):
    """Get tasks by space token"""
    url = f'https://unpod.dev/api/v2/platform/spaces/{space_token}/tasks/'

    headers = {
        'Authorization': 'JWT your-jwt-token',
        'Content-Type': 'application/json'
    }

    params = {'page': page, 'page_size': page_size}

    response = requests.get(url, headers=headers, params=params)
    data = response.json()

    print(f"Total tasks: {data['count']}")
    return data['data']

def create_task(space_token: str, pilot_handle: str, documents: list, context: str):
    """Create a new task in a space"""
    url = f'https://unpod.dev/api/v2/platform/spaces/{space_token}/tasks/'

    headers = {
        'Authorization': 'JWT your-jwt-token',
        'Content-Type': 'application/json'
    }

    payload = {
        'pilot': pilot_handle,
        'documents': documents,
        'context': context,
        'schedule': {'type': 'now'}
    }

    response = requests.post(url, headers=headers, json=payload)
    data = response.json()

    print(f"Task created: {data['data']['run_id']}")
    return data['data']

# Example usage
fetch_tasks_by_agent('sales-agent-v1')
```

## Best Practices

1. **Pagination**: Use pagination parameters for spaces with many tasks to improve performance
2. **Agent Handle**: Use meaningful and consistent agent handles for better task organization
3. **Context**: Provide clear context in task creation for better execution outcomes
4. **Error Handling**: Always handle potential errors and implement retry logic
5. **Document Structure**: Include all relevant contact information in documents
6. **Scheduling**: Use `schedule.type: "now"` for immediate execution or configure for later
7. **Security**: Keep JWT tokens secure and rotate them regularly
8. **Monitoring**: Track task completion rates and agent performance metrics
