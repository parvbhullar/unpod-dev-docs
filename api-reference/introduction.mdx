# Unpod API Reference

Welcome to the Unpod API reference documentation. This document provides detailed information about our REST API, including available endpoints, request/response formats, and code examples.

## Base URL

All API requests should be made to one of the following base URLs:

- **Production**: `https://api.unpod.dev/v1`
- **Sandbox**: `https://sandbox.api.unpod.dev/v1`

## Authentication

Unpod uses API keys for authentication. Include your API key in the `Authorization` header:

```
Authorization: Bearer YOUR_API_KEY
```

[Get your API key](https://app.unpod.dev/settings/api-keys)

## Rate Limiting

API requests are subject to rate limiting:

- **Standard Rate Limit**: 100 requests per minute per API key
- **Burst Limit**: 10 requests per second

## Response Format

All API responses are in JSON format and include standard HTTP status codes.

## Error Handling

Errors return a JSON object with an `error` field containing details:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "The request was invalid.",
    "details": {
      "field": "phone_number",
      "reason": "invalid_format"
    }
  }
}
```

## API Endpoints

Explore the available API endpoints using the navigation menu. The API is organized into the following sections:

- **Authentication**: Manage API keys and authentication
- **Numbers**: Search, purchase, and manage phone numbers
- **Calls**: Make and manage phone calls
- **Messages**: Send and track SMS/MMS messages
- **Webhooks**: Configure and manage webhook endpoints
