## What Is an Agent?

An **agent** in Unpod ties together a voice profile (STT + TTS), telephony (phone numbers), and your runner logic into a single deployable unit. When a call arrives on a number attached to the agent, the orchestrator dispatches it to your `AgentRunner` process.

```
  Phone Number
       │
       ▼
    Agent  ──── voice_profile ──► STT/TTS stack
       │
       ▼
  AgentRunner (your Python process)
       │
       ▼
   CallContext → entrypoint()
```

## Creating an Agent

```python
import asyncio
from unpod import AsyncClient

async def main():
    async with AsyncClient(api_key="sk-...") as client:
        agent = await client.agents.create(
            name="Support Bot",
            voice_profile="vp_en_female_hd",    # from voice_profiles.list()
            recording=True,                      # store call recordings
            max_call_duration_s=600,             # 10-minute hard cap
        )
        print("Created:", agent.agent_id, agent.name)

asyncio.run(main())
```

### `create()` parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `name` | `str` | required | Display name for the agent |
| `voice_profile` | `str \| None` | `None` | Voice profile ID (`vp_...`) |
| `runner_agent_id` | `str \| None` | `None` | Override the runner worker target |
| `agent_endpoint` | `str \| None` | `None` | HTTP endpoint for the dialog brain (HTTPAdapter) |
| `recording` | `bool` | `False` | Enable call recording |
| `max_call_duration_s` | `int` | `3600` | Hard cap in seconds (default 1 hour) |

## Listing Agents

```python
import asyncio
from unpod import AsyncClient

async def main():
    async with AsyncClient(api_key="sk-...") as client:
        agents = await client.agents.list()
        for a in agents:
            print(a.agent_id, a.name, a.voice_profile_id or "no voice profile")

asyncio.run(main())
```

### Agent fields

| Field | Type | Description |
|---|---|---|
| `agent_id` | `str` | Agent ID (`agt_...`) |
| `name` | `str` | Display name |
| `voice_profile_id` | `str \| None` | Voice profile ID |
| `agent_endpoint` | `str \| None` | HTTP dialog endpoint |
| `recording` | `bool` | Recording enabled |
| `max_call_duration_s` | `int` | Call duration hard cap |
| `created` | `datetime` | Creation timestamp |
| `modified` | `datetime` | Last modified timestamp |

            "agt_...",

## Deleting an Agent

<Warning>
Deleting an agent does **not** automatically detach phone numbers. Detach all numbers before deleting to avoid orphaned routing.
</Warning>

```python
import asyncio
from unpod import AsyncClient

async def main():
    async with AsyncClient(api_key="sk-...") as client:
        # Detach numbers first
        numbers = await client.numbers.list()
        for n in numbers:
            if n.agent_id == "agt_...":
                await client.numbers.detach(n.id)

        # Then delete
        await client.agents.delete("agt_...")
        print("Deleted")

asyncio.run(main())
```

## Using an Agent with an HTTP Dialog Endpoint

If you already have a chatbot or API (in any language) and want to make it voice-enabled, point the agent at an HTTP endpoint. Unpod calls your endpoint on every user turn, you return the reply.

```python
import asyncio
from unpod import AsyncClient

async def main():
    async with AsyncClient(api_key="sk-...") as client:
        agent = await client.agents.create(
            name="Webhook Bot",
            voice_profile="vp_en_male_std",
            agent_endpoint="https://your-api.example.com/dialog/turn",
        )
        print("Agent with HTTP endpoint:", agent.agent_id)

asyncio.run(main())
```

### HTTP Protocol

Unpod sends a `POST` request on every user turn:

```
POST https://your-api.example.com/dialog/turn
Content-Type: application/json

{
  "text": "I'd like to reschedule my appointment",
  "context": {
    "call_id": "call_abc123",
    "session_id": "sess_xyz789",
    "turn_number": 3,
    "direction": "inbound",
    "user_number": "+919876543210",
    "data": {}
  }
}
```

Your endpoint must respond with:

```json
{
  "text": "Sure, I can help with that. What date works for you?"
}
```

<Note>
This protocol is **not** OpenAI-compatible. It is a simple text-in / text-out interface — your endpoint owns all LLM or chatbot logic. You can forward to OpenAI, Gemini, a rules engine, or anything else under the hood.
</Note>

Alternatively, use the `HTTPAdapter` inside an `AgentRunner` entrypoint to call a remote endpoint from within the SDK:

```python
from unpod import AgentRunner, CallContext
from unpod.adapters.http import HTTPAdapter

async def handle_call(ctx: CallContext) -> None:
    ctx.session.dialog_machine = HTTPAdapter(
        url="https://your-api.example.com/dialog/turn",
    )
    await ctx.session.run()

AgentRunner(entrypoint=handle_call, agent_id="my-bot").start()
```

## Agent + Runner Pattern (Recommended)

The recommended pattern for production is a dedicated `AgentRunner` process:

```python
from unpod import AgentRunner, CallContext

AGENT_ID = "agt_..."   # must match what you created above

async def handle_call(ctx: CallContext) -> None:
    await ctx.session.say("Hello, how can I help you?")
    await ctx.session.run()

runner = AgentRunner(
    entrypoint=handle_call,
    agent_id=AGENT_ID,
    max_sessions=20,
)
runner.start()
```

The `agent_id` in `AgentRunner` must match the agent's ID in the platform. The orchestrator uses it to route dispatches to the correct runner worker.
